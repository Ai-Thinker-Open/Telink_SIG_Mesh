1, 不要使用 #ifdef, 在 c 文件中尽量使用 if, 在 头文件中尽量使用 #if. 除了以下几种情况
	A, 头文件的开始的防御宏  #ifndef xxxx
	B, 编译器预设的宏。 譬如 #ifdef WIN32,  #ifdef __GNUC__ 等
	
2, 命名规则：
	A, 每个模块一个文件，模块名为名词，通常不使用复数、时态等语法。
		模块内函数变量开头名字跟文件名对应
	B, 如果命名很难或者不合适， 那么更改模块名，或者新建一个模块
	c, 命名除了宏外，其他都用小写加下划线组成。
	D, 函数命名是 modulename_动词_xxxxx
3, 头文件包含
	A, 头文件包含尽量最小化，但是要完备；就是头文件本身已经包含了所有需要的其他头文件，但是不会更多。
	B, 一般来说头文件要先包含基础部分，然后才是功能部分，但不强制。包含顺序，不应该影响软件行为，

4, 宏定义
	A，尽量少使用宏，应该使用 const 变量定义，单不强制
	B, const 变量有如下限制，所以不能完全替代宏。
		const 变量不能用于 case xxx 中作为条件分支
		const 只能用于c源文件， 不能用于头文件中做为条件分离代码
